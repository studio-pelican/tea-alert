# ウォークスルー: ツイキャスギフトアラートの導入と使用方法

実装・ビルドがすべて完了しました！
この機能は、OBS Studioのブラウザソースとして組み込める「ツイキャスギフト通知アニメーション」です。

## 完了した実装内容

1. **設定画面 (`/`)**: 
   - 配信者の `screen_id` とアクセストークンを入力し、セキュアにブラウザ（`localStorage`）へ保存する機能を実装しました。
   - OBS上での実際の表示を確認するための **「ダミー通知ボタン」** を配置しました。
2. **アラートオーバーレイ (`/alert`)**:
   - 背景を完全に透明化（透過）し、OBS上に綺麗にオーバーレイされるように設計しました。
   - Framer Motion を使用し、ギフト受信時のポップアップや、アイテム画像が跳ねるアニメーション、光芒エフェクト（Glow）を実装しました。
   - 5秒経過で自動的にフェードアウトし、キューから削除されます。
3. **APIポーリング**:
   - `https://apiv2.twitcasting.tv/users/{screen_id}/gifts` エンドポイントに対して5秒間隔でリクエストを送信。
   - `slice_id` を用いて、新規ギフトのみを差分取得するロジックを実装しました。

## ローカル環境でのテスト手順

1. ターミナルで `npm run dev` を実行し、開発サーバーを起動します。
3. まず、Step 1で **「ツイキャスで連携する」** ボタンをクリックし、ツイキャスの認証画面を開きます。
4. 認証を許可すると、設定画面に戻ってきて自動的にStep 2へ進みます。
5. Step 2の画面で、ご自身の `screen_id` （例: `pelicamizudori` や任意のテスト用ID）を入力し、「設定を保存して完了」をクリックします。
6. **「アラート画面 (OBS用) を開く」** リンクをクリックし、別タブで `/alert` ページを開きます。
6. 設定画面に戻り、**「ダミー通知を送信」** ボタンをクリックします。
7. `/alert` のタブを見ると、ダミーの「お茶 (10MP)」のギフトがポップアップアニメーションとともに表示され、5秒後に消えることが確認できます。

## OBS Studio への組み込み設定

1. OBS Studio を起動し、「ソース」ブロックの `+` ボタンから **「ブラウザ」** を追加します。
2. ソースのプロパティを以下のように設定します。
   - **URL**: `http://localhost:3000/alert` (※本番デプロイ後はデプロイ先のURL、例: `https://your-github-username.github.io/tea-alert/alert` など)
   - **幅**: `1920`
   - **高さ**: `1080`
   - **カスタムCSS**: （デフォルトのまま。`body { background-color: rgba(0, 0, 0, 0); margin: 0px auto; overflow: hidden; }`）
   - [オプションのチェック]: 「ソースがアクティブにならないときは表示しない」
3. 設定を保存すると、OBSのプレビュー画面に透明なオーバーレイが重なります。

## GitHub Pages へのデプロイについて

このプロジェクトは Next.js 15 の **Static Exports**（`output: "export"`）に完全対応しています。

1. `npm run build` コマンドで `out/` ディレクトリが生成されます。
2. 生成された `out/` ディレクトリの中身をホスティングサーバーやGitHub Pagesにアップロードするだけで動作します。

## トラブルシューティング

- **通知が出ない・APIエラーが出る**
  - アラート画面の左上に赤いエラーボックスが表示されます。主に「トークンの期限切れ（401 Unauthorized）」などのエラーですので、再度APIページでトークンを取得し直して設定画面から再設定してください。
- **OBS上で背景が黒（または白）になってしまう**
  - OBSのブラウザソースプロパティの「カスタムCSS」に `body { background-color: rgba(0, 0, 0, 0); }` が含まれているか確認してください。
