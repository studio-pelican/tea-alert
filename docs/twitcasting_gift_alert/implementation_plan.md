# 実装計画: ツイキャスギフトアラートWebアプリ

## 要件理解要約

- **目的**: OBS Studioのブラウザソースとして機能する、ツイキャスのギフト通知リアルタイムWebアプリの構築。
- **機能**: 
  - 配信者の `screen_id` とAPIの `token` を入力し、設定を保存（ブラウザの `localStorage`）。
  - ツイキャスAPI V2 (`/gifts`) を5秒間隔でポーリングし、新規ギフトを検知。
  - 新規ギフトがある場合、透明背景のアラートアニメーション（ポップアップ、スケール、光るエフェクト付き）を表示し、5秒後に自動フェードアウト。
- **技術要素**: Next.js 15, TypeScript, Tailwind CSS, Framer Motion。GitHub Pagesへのデプロイを想定。
- **UI/UX**: ダークベースのグラデーション、配信映えする「プロダクション品質」。CORS対応済み。

## 提案アーキテクチャ

- **アプリケーション構成**:
  - **SPA形式**: GitHub Pagesへの静的ホスティングを見据え、Static Export 機能 (`output: 'export'`) を利用。
  - **ルーティング**:
    - `/` または `/settings`: 初回アクセス時の設定画面（`screen_id`、`token` 入力フォーム・ダミーテスト発火ボタン）。
    - `/alert`: OBSのブラウザソース用アラート表示画面（1920x1080、要素外は完全透明 `background: transparent`）。
- **データフロー**:
  1. ユーザーが設定画面で入力・保存（localStorage: `twitcast_settings`）。
  2. OBSのブラウザソースで `/alert` にアクセスし、ストレージから設定を読み込み。
  3. 初回マウント時、`slice_id = -1` で `/gifts` リクエストを送信し、最新の `slice_id` を取得。
  4. 以降5秒周期(`setInterval`) で最新の `slice_id` を用いてポーリング。
  5. 新たなギフトが取得できれば、Reactのステート（キュー）に追加し、Framer Motion を用いた `AnimatePresence` で通知コンポーネントをレンダリング。表示後5秒経過したイベントはキューから削除。
